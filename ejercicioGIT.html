  1. ¿Que es un sistema de control de versiones?

    Es una forma de poder poner en común varias versiones o partes diferentes de un proyecto para posteriormente fusionarlas o mejorarlas.

  2. ¿Cual es la diferencia entre git y github?

    Git es un sistema de control de versiones el cual tiene sus comandos predeterminados con los que se pueden realizar diferentes acciones con los proyectos mientras que GitHub es una plataforma donde se pueden subir o modificar los proyectos en repositorios usando Git.

  3. ¿Cómo creo una rama en git?

    Para crear una nueva rama hay que usar la orden "git branch nombre" indicando el nombre que queremos darle a la rama.

  4. ¿Cómo clono un repositorio?

    Un repositorio se puede clonar usando la orden "git clone url" donde url es la url del repositorio a clonar que puede ser en SSH o HTTPS.

  5. ¿Cómo puedo bajar los ultimos cambios de un repositorio?

    Para bajar los últimos cambios de un repositorio en GitHub por ejemplo a local podemos usar el comando "git pull" desde la terminal con la dirección del proyecto que queremos actualizar.

  6. ¿Cómo puedo generar un tag?

    Para generar un tag se puede hacer usando la orden "git tag nombre" donde el nombre es aquel que queremos darle al tag.

  7. ¿Cómo hago una mezcla de ramas?

    Para hacer una mezcla de ramas hayque utilizar la orden "git merge nombre" donde en el nombre hay que indicar el nombre de la rama que queremos fusionar con la rama actual.

  9. ¿Cómo puedo borrar una rama?

    Para eliminar una rama usaremos la opción -d del comando git branch resultando de la siguiente forma "git branch -d nombre" donde el nombre es aquel de la rama que queremos borrar.

  10. ¿Cómo puedo borrar un tag?

    Para eliminar un tag creado anteriormente usaremos la orden "git tag -d nombre" indicando el nombre del tag que queremos borrar.

  11. ¿Cómo puedo ver los cambios que estan por enviarse al repositorio?

    Para ver los cambios que están por enviarse al repositorio lo podrenmos hacer usando la orden "git status". En la terminal nos aparecerá en rojo aquellos ficheros modificados o añadidos que aún no han sido enviados.

  12. ¿Para que sirve el comando git remote -v?

    El comando "git remote -v" muestra las URLs de los repositorio remotos.

  13. ¿Cómo le digo a git cual es mi email y correo?

    Para configurar tu nombre y correo en Git debemos usar git config --global user.name "Tu Nombre" y git config --global user.email "tuemail@example.com".
